## 8. 已知 caveats

1. **RoboHire `/parse-resume` 和 `/match-resume` 当前 100% 返回 HTTP 500**（他们后端配置使用的 OpenRouter 模型 ID `openrouter/google/gemini-3.1-pro-preview` 不存在）。我们用 LLM 网关（new-api `google/gemini-3-flash-preview`）替代，**输出 schema 和 RoboHire response 完全一致**，等他们修好把 `extractResumeFromBuffer` 切回 RoboHire 客户端就行，下游 RAAS 完全无感。详见 [docs/e2e-real-pdf-test-report.md](e2e-real-pdf-test-report.md)。

2. **JD 当前是从文件名 regex 推断的**（`【职位_城市 薪资】候选人 工作年限.pdf`），是 PoC 占位。等 RAAS 提供 `job_requisition_id → JD 文本` 接口（HTTP fetch）就替换 [match-resume.ts:103-112](../server/ws/agents/match-resume.ts#L103-L112) 那一段。

3. **Inngest UI 上看到的 `Resume Parser Agent` Failed**：那是旁边老 prototype 项目（`~/Desktop/agenticOperator/resume-parser-agent/`）的 fn，期待旧 schema 所以 throw —— 我们的 `processResume (workflow node 9-1)` 同一时刻 Completed。要清理就 `kill $(lsof -ti :3020)`。
